## Known gap (pre-existing, deferred to v0.7)
- **CLI entry point only exposes the web scanner.** `pyproject.toml`/`setup.py` map the
  `gitexpose` console script to `gitexpose.cli:main` (web-scan only). The advanced command
  group (`gitexpose.cli_advanced:cli` — `scan`, `supply-chain`, `git-history`, `agent-audit`,
  …) is reachable only via `python -m gitexpose.cli_advanced <cmd>`. The README examples
  (`gitexpose agent-audit`, `gitexpose supply-chain`, `gitexpose git-history`) therefore don't
  match the installed binary. This predates v0.6 (true since v0.4/v0.5). v0.7 should unify the
  entry point — likely repoint `gitexpose` → the `cli_advanced` group (web-scan becomes
  `gitexpose scan <target>`), and update the README accordingly.
