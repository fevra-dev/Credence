If we rename — the migration cost (eyes open)

  1. PyPI package — new distribution name; or a final gitexpose release pointing to the successor.
  2. Console entry points — pyproject.toml [project.scripts] → new binaries; keep gitexpose as a deprecated alias for one minor.
  3. Repo / GitHub — rename (auto-redirects), update badges, CI workflow names, release assets.
  4. Docs — README header/tagline, CHANGELOG, COVERAGE.md, sample Action, install snippets.
  5. Internal package dir — keep gitexpose/ for v0.8; rename brand + CLI + PyPI only, to avoid import churn across every test.
  • PyPI distribution name free
  • GitHub repo / org name free
  • No major security-tool collision ("<name> security scanner")
  • ASCII CLI binary clean, no shell-builtin clash
  • Domain optional (.dev / .io)

Hold Credence as the standalone fallback if expose collides on PyPI, and Aperture as the design-forward third.
