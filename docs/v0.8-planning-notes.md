# GitExpose v0.8 — Planning Notes

Shipped v0.8.0 "AI-Infra Layer, Deepened": four high-precision, stdlib-only AI-infrastructure
finding capabilities plus a shared severity exit gate, positioned to **run alongside**
TruffleHog/Gitleaks (the SARIF `partialFingerprints` are the mechanical cross-tool dedup proof).

## Shipped in v0.8
- **`--fail-on {info,low,medium,high,critical}` severity exit gate** (default `high`) across
  `supply-chain` / `agent-audit` / `git-history`. Behavior change: previously any finding exited
  non-zero; now only HIGH/CRITICAL do by default. `--fail-on info` restores the old behavior.
- **Git-metadata credential spine** (`supply-chain`): `git_config_credential_url`,
  `git_config_extraheader_credential`, `gitmodules_credential_url`, `git_config_generic_token_url`.
  `configparser`-only, **CVE-2025-41390-safe** (never invokes git on the target).
- **Agent debug-print AST detector** (`agent-audit`): `agent_skill_credential_print` — flags
  `print()`/`logging.*()` of credential-named variables (not string literals) in agent/skill Python.
- **MCP security posture score (0-100)** (`agent-audit`): decoupled per-issue findings
  (`mcp_static_credential`, `mcp_plaintext_http`, `mcp_unknown_origin`, `mcp_unpinned_version`) with
  honest gating severities + an INFO `mcp_server_posture` summary carrying the score. Env-var
  passthrough (`${VAR}`) is not flagged as a static credential.
- **Orphan cross-source signal** (`supply-chain`, opt-in `--track`/`--registry`): hash-only
  `SecretRegistry`, `source_frequency` band, `KNOWN_EXAMPLE_KEYS` downgrade, and SARIF
  `partialFingerprints["secretValueHash/v1"]` for cross-tool dedup.
- **`supply-chain --output sarif`**: supply-chain findings now export SARIF 2.1.0.
